Centos7.1 ssh端口修改

最近项目上发觉服务器ssh 22端口经常被扫描，因此写了centos下修改ssh端口方法。

1.修改/etc/ssh/sshd_config文件

vi /etc/ssh/sshd_config

Port 22 这行去掉#号，防止配置不好以后不能远程登录，还得去机房修改，等修改以后的端口能使用以后在注释掉；

Port 8788 下面添加你想要的端口；

2.修改防火墙配置

2.1firewall防火墙

firewall添加想要修改的ssh端口：
firewall-cmd –zone=public –add-port=20000/tcp –permanent (permanent是保存配置，不然下次重启以后这次修改无效)

reload firewall：

firewall-cmd –reload

查看添加端口是否成功，如果添加成功则会显示yes，否则no

firewall-cmd –zone=public –query-port=8788/tcp

2.2 iptables 防火墙

注意网上有说卸载iptables方式，yum remove iptables，千万不要执行这个指令，否则你会悲剧

先查看iptables原有规则:

1、iptables -nvL

2、more /etc/sysconfig/iptables

3、加iptables规则

-A INPUT -p tcp -m tcp –dport 22 -j ACCEPT

-A INPUT -p tcp -m tcp –dport 8788 -j ACCEPT

-A INPUT -p tcp -m state –state NEW -m tcp –dport 8788 -j ACCEPT

3.修改SELinux

3.1 安装

yum install policycoreutils-python

3.2 查看当前SElinux 允许的ssh端口

semanage port -l | grep ssh

添加你需要修改的端口:semanage port -a -t ssh_port_t -p tcp 8788

然后确认一下是否添加进去:semanage port -l | grep ssh

如果成功会输出:
ssh_port_t tcp 8788, 22

4.重启ssh服务

systemctl restart sshd.service

5.测试ssh是否能连上

测试修改端口以后的ssh连接，如果成功则将step1里面的port 22 重新注释掉
